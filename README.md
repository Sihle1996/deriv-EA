# deriv-bot — data spine + ATS Master Pattern research

A demo-only Deriv synthetic-indices research harness. It connects to the Deriv WebSocket API,
streams **ticks + one base candle series**, resamples higher timeframes in-process, persists
**every tick** to Parquet (gap-free across reconnects), and runs the **ATS "Forex Master Pattern"
detector** (value line + HTF→LTF pullback entry) that **logs candidate signals** for offline review.
**No trading** — execution is a later phase.

> ⚠️ **Reality check.** Deriv synthetic indices are generated by a cryptographically secure RNG.
> There is no order flow, news, or liquidity to read, so chart patterns do not carry a predictive
> edge here. This project is built as honest infrastructure and a research harness — not a
> money-printing machine. Phase 2's whole point is to **measure** whether signals have any forward
> edge (via a null model); the expected, correct answer on an RNG is *no edge*. Start (and stay,
> for now) on a **demo** account.

## Requirements

- Python 3.11+ (developed on 3.13)
- **No API token needed for Phase 1.** On the legacy Deriv WebSocket API, market data
  (ticks / candles / `active_symbols`) is **public** — `authorize` is only required for account
  calls (balance, buy, portfolio), which arrive in later phases.

## Setup

```bash
python -m venv .venv
.venv\Scripts\activate        # Windows PowerShell:  .venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

That's it — `python main.py` runs immediately against public synthetic-index data.

### Authentication (later phases only)

Phase 1 runs **unauthenticated** (`DERIV_AUTHENTICATE=false`, the default). When we add
account/trading features you'll set `DERIV_AUTHENTICATE=true` and provide a token.

> **Note on Deriv's two API systems.** Newer Deriv accounts only mint **`pat_`-prefixed
> Personal Access Tokens**, which belong to Deriv's *new* Options API (a REST OTP → WebSocket-URL
> flow on `api.derivws.com`). Those tokens are **rejected** by this legacy v3 `{"authorize": …}`
> call. The trading phase will implement the OTP flow for the `pat_` token; the legacy data
> feed used here needs no token at all. When authentication is enabled, the app
> **hard-blocks real-money accounts** (exits unless `is_virtual` is true).

## Run

```bash
python main.py           # stream + persist + LOG ATS signals; Ctrl-C to stop (flushes everything)
python verify_feed.py    # feed-quality checks (no token needed)
python check_archive.py  # audit the persisted tick archive for gaps/continuity
python verify_ats.py     # ATS detector regression (offline, deterministic)
python review_signals.py # forward outcomes + null-model comparison
python backfill_signals.py --dry-run  # regenerate the complete ATS signal set from the tick archive
python backtest_signals.py            # simulate Rise/Fall trades + P&L vs a null model (the money question)
python validate_signals.py            # honest stats: permutation p-value, walk-forward, PBO/CSCV, deflated Sharpe
python verify_validation.py           # unit-tests the validation math on known-truth inputs
```

`main.py` persists every tick **and**, on each candle close, feeds the snapshot to the ATS detector
and logs any signal to `data/signals_ats/<symbol>/<date>.jsonl`. Lines look like
`ATS 1m entry up^  value=7992.10 price=7991.30 bias=up`.

On reconnect or restart, `main.py` automatically **backfills** ticks missed during the outage (via
`ticks_history`), so the archive stays gap-free for backtesting — within a ~83-minute window per
outage. On startup it also prints every `step`-matching symbol from `active_symbols` so you can
confirm the exact Step Index code rather than trusting the `stpRNG` default.

### The ATS Master Pattern logic

A faithful encoding of the TradeATS "Forex Master Pattern" (the contraction rule is code-audited from
LuxAlgo's open-source Pine), pairing a higher timeframe (HTF, default 15m) that sets directional bias
with a lower timeframe (LTF, default 1m) that gives the entry:
- **Contraction**: a **swing-pivot compression** — a confirmed pivot-high lower than the prior
  pivot-high AND a pivot-low higher than the prior pivot-low, over `ats_pivot_lookback` bars each side.
  Freezes a **value line** at the box midpoint `(top+btm)/2`, projected forward as a reference level.
- **Expansion**: a plain close break of a box boundary (audited: no ATR buffer).
- **Entry**: on the LTF, price **pulls back to its value line** (within `ats_pullback_tol_atr × ATR`)
  in the breakout direction — and the entry fires **only if it agrees with the HTF bias** (HTF close
  vs its own value line). Gated pullbacks are logged as `entry_blocked` for the funnel. Params in `config.py`.

> The contraction/value-line/expansion rules are verifiable from open-source indicators; entry/stop/
> target rules are NOT in any public source (TradeATS paid-course material), so the pullback entry is
> our own heuristic. And on a CSPRNG synthetic no edge is possible by construction — the detector is
> faithful so the harness can *measure* that honestly, not to imply profit.

`review_signals.py` joins each logged signal against the archived ticks (look-ahead firewall:
only ticks with `epoch > bar_close_epoch`), computes forward return / MFE / MAE / first-touch, and
compares to a **null model** of random entries. If real ≈ random, there's no edge.

## Layout

| File | Role |
|------|------|
| `config.py` | env-driven config (symbol, timeframes, paths, reconnect, signal params) |
| `deriv_client.py` | raw-websockets client + auto-reconnect/resubscribe + tick backfill |
| `candles.py` | `MultiTimeframeStore` + `MarketSnapshot` (the detector surface); `TFView`/`closed_frame`/`_compute_view` (all pandas/indicator math lives here — ATR + inside-bar contraction box) |
| `ats_signals.py` | pure ATS detector — `AtsTimeframeDetector` (per-TF state machine + value line) + `AtsEngine` (HTF-bias-gated LTF entries), no pandas/IO |
| `signals.py` | `SignalRecord` dataclass + schema version (the logged unit; no detector) |
| `storage.py` | `TickStore` → daily Parquet; `SignalStore` → daily JSONL (`data/signals_ats/<symbol>/<date>.jsonl`) |
| `main.py` | wire-up: seed-from-archive → connect → subscribe → persist → snapshot-on-close → ATS detect+log → reconnect/restart backfill |
| `verify_feed.py` | feed checks: alignment / gap / resample-fidelity / tick-archive |
| `check_archive.py` | audits the persisted tick archive for gaps/continuity |
| `verify_ats.py` | ATS detector regression (contraction/value-line/breakout/pullback/HTF-gate/timeout/dedup) |
| `review_signals.py` | outcome measurement: forward returns vs a null model |
| `backfill_signals.py` | regenerate the complete signal set from the gap-free tick archive (recovers any signals missed during outages; deduped vs the live log) |
| `backtest_signals.py` | contract-economics backtester — simulates Rise/Fall trades from signals, applies the payout haircut, reports P&L/ROI vs a null model (answers "would it make money?") |
| `validate_signals.py` | honest statistical validation — Monte-Carlo permutation test, walk-forward IS/OOS, PBO via CSCV, deflated Sharpe (answers "is any edge real or overfit luck?") |
| `verify_validation.py` | unit-tests the validation statistics on known-truth inputs |

## Dashboard (read-only research viewer)

A React dashboard (live candles + signal markers, recent-signal log, backtest P&L verdict, archive
health) backed by a FastAPI server. **Read-only — no trading.** The backend opens its OWN public
Deriv connection for real-time charts (independent of the collector bots) and reads
signals/backtest/health from the files the bots write.

```bash
# one-time: backend deps + frontend deps
.venv\Scripts\python -m pip install -r dashboard\requirements.txt
cd dashboard\web && npm install && cd ..\..

# dev (two terminals): backend, then the Vite dev server (proxies /api + /ws to :8000)
.venv\Scripts\python -m uvicorn dashboard.server:app --port 8000
cd dashboard\web && npm run dev      # open the printed http://localhost:5173

# OR production (one process): build the SPA, then uvicorn serves it at /
cd dashboard\web && npm run build && cd ..\..
.venv\Scripts\python -m uvicorn dashboard.server:app --port 8000   # open http://localhost:8000
```

Layout: `dashboard/server.py` (FastAPI REST + WS), `dashboard/live.py` (per-symbol live feed,
reuses `deriv_client`/`candles`), `dashboard/readers.py` (signals/backtest/health from files),
`dashboard/web/` (Vite React+TS + lightweight-charts). The dashboard discovers symbols from
`data/ticks/*`, so it shows whatever the bots are collecting.

## Phase gates

- **Data spine — PASSED.** 24h soak: 100% gap-free archive, `verify_feed.py` 4/4.
- **ATS Master Pattern research — current.** Collect signals with `main.py`, review with
  `review_signals.py` / `validate_signals.py`. Still **no trading**.
- **Gate before trading:** > 500 reviewed signals **and** `validate_signals.py` showing a
  demonstrated, repeatable edge — permutation **p < 0.05**, out-of-sample survival, **low PBO**, and a
  best-config Sharpe above the deflated/expected-max hurdle. On a CSPRNG synthetic the expected result
  is *no edge* → stay in research. That negative result is the point. No trade-execution code is
  written until this gate is cleared. (A deep-research pass — 25 primary-sourced claims, 0 refuted —
  confirmed neither SMC nor the Forex Master Pattern has verifiable edge, and that on RNG synthetics
  it's impossible by construction; the honest improvement is rigorous testing, not a better pattern.)
